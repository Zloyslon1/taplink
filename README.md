# Микросайт-витрина `/taplink/`

Single-page страничка для шапки Instagram `@mosunova.rod`. Заменяет «таплинк-список из 5 кнопок» полноценной упаковкой эксперта. Лежит в основном репо `pages` (рядом со статьями), но **не использует сборку и не зависит от React/esbuild/Tailwind** — это plain HTML/CSS/JS, render отдаёт `index.html` и картинки как есть.

## URL

**Боевой:** `https://pages-gmth.onrender.com/taplink/`

Раздаётся render автоматически после `git push origin main` (через 1–3 минуты). Никакой сборки, никаких артефактов.

## Структура страницы

- **Шапка** — аватар, имя, 4 строки регалий/соц.доказательства. Без внешних ссылок (ifpp.ru убрали 06.05). Текст по левому краю.
- **Таб «Тесты»** — главный баннер «Точка Разворота» (интенсив, LCP) + 3 карточки тестов-воронок (ИГРА, ЛЮБОВЬ, МАМА). Заголовков-дублей с JPEG-обложек нет — на картинке уже крупный заголовок, под ней только описание.
- **Таб «Курсы»** — каталог из 2 продуктов с зачёркнутыми старыми ценами (5000→0₽, 3000→0₽). `<h3>` заголовки убраны (тоже на картинках), `course__desc` стилизован как лид.
- **Таб «Каналы»** — Telegram + YouTube с осмысленными подписями (не «Telegram-канал»/«YouTube-канал»).

Футера-копирайта нет (убрали 06.05).

Дизайн: белые парящие карточки на голубоватом фоне `#F1F5FC`, акцент `#1F4FD1`, тени `0 4px 14px rgba(15, 50, 140, .08)`. Mobile-first, max-width 480px, **системный шрифт-стек** (Inter сняли ради perf, см. ниже).

**Табы — стиль «liquid glass»** (с 06.05.2026, v1.0.3): pill-контейнер с `backdrop-filter: blur(20px) saturate(180%)`, полупрозрачный белый фон, активная вкладка — белый pill с внутренним хайлайтом и мягкой тенью. До этого был underline border-bottom со scrollable полоской — поменяли после показа 4 вариантов.

## Локальная разработка

```bash
python3 -m http.server 8765
# открыть http://127.0.0.1:8765/taplink/
```

`file://` тоже работает, но абсолютные пути (`/taplink/images/...`) будут резолвиться как путь к корню FS — лучше через сервер.

После правок:
```bash
git add taplink/
git commit -m "..."
git push origin main
```

Render авто-задеплоит за 1–3 минуты.

## Принципы (зачем именно так)

### Без сборки

В отличие от `articles/` (esbuild + Tailwind CLI), здесь сборки нет — render просто раздаёт `index.html`. Это сознательно: страница маленькая, без зависимостей, добавлять билд-flow ради двухсот строк CSS — overhead. Если когда-то понадобится JSX/React — заводить отдельную папку `src/taplink/` и build-таргет в `build.mjs`.

### Абсолютные пути к статикам

Все `src="/taplink/images/..."` и `href="/taplink/..."` — **только абсолютные**. То же правило, что для статей (см. PROJECT_BRIEF.md раздел 11.9). Render rewrite-правила в дашборде могут отдавать страницу по разным URL-формам, относительные пути ломаются.

### Производительность

История: первый деплой выдал FCP 2.6с / LCP 3.5с (Lighthouse, эмуляция Moto G Power, 4G slow). Стало FCP 0.8с / LCP 1.1с после прохода 05.05.2026:

- **Снят Google Fonts (Inter, 4 веса)** — был render-blocking, экономия ~1840 мс. Шрифт теперь `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` — кириллица отлично рендерится на iOS/Android.
- **Убран `background-attachment: fixed`** на `<body>` — на iOS он вызывал repaint при каждом скролле (отсюда было ощущение лагов).
- **Картинки пережаты** через `sips -Z <ширина> -s formatOptions 82`: 772 KB → ~360 KB. Целевая ширина — 2× retina от показа (960px для 480px-баннеров, 320px для 112px-карточек, 192px для 96px-аватара).
- **Атрибуты `width`/`height`** на всех `<img>` (стабильный layout).
- **`loading="lazy"`** на 7 картинках ниже первого экрана (3 карточки тестов в табе «Тесты» — спорно, но они короткие и грузятся за миллисекунды; все картинки в табах «Курсы» и «Каналы»).
- **`decoding="async"`** на всех картинках.
- **`fetchpriority="high"`** на аватаре и главном баннере (LCP-кандидаты).
- **`touch-action: manipulation` + `-webkit-tap-highlight-color: transparent`** на ссылках и кнопках — мгновенная реакция на тап без 300мс-задержки.

### Безопасность

В HTML на уровне meta:
- **CSP** (`default-src 'self'; img-src 'self' data:; style-src 'self' 'unsafe-inline'; script-src 'self' 'unsafe-inline'; ...; form-action 'none'; base-uri 'self'; object-src 'none'; frame-src 'none'; upgrade-insecure-requests`) — запрет любых внешних скриптов/стилей/iframe/object и сабмита форм.
- **Referrer-Policy** = `strict-origin-when-cross-origin` — не сливать полный URL в Referer на t.me/ifpp-web/youtube (происхождение остаётся для аналитики).
- **`rel="noopener"`** на всех 9 внешних ссылках — антидот к tabnabbing на случай нештатного поведения WebView.

На уровне HTTP-заголовков (через meta их по спецификации нельзя — нужен Render dashboard, секция «Headers», path `/taplink/*` или глобально):

| Header | Value |
|---|---|
| `Strict-Transport-Security` | `max-age=31536000; includeSubDomains` |
| `X-Content-Type-Options` | `nosniff` |
| `X-Frame-Options` | `DENY` |
| `Cross-Origin-Opener-Policy` | `same-origin` |
| `Permissions-Policy` | `camera=(), microphone=(), geolocation=(), payment=()` |

Эти 5 заголовков закроют пункты Lighthouse про COOP-изоляцию, кликджекинг и HSTS. Через render-yaml не делал — в репо живёт большая статья-воронка, а render.yaml перезаписал бы dashboard-конфиг и мог сломать rewrite-правила для статей.

## Аналитика (с v1.0.8, 07.05.2026)

На странице стоит лёгкий анонимный трекинг просмотров и кликов. Шлёт в тот же Apps Script, что и статьи (`shared/apps-script.gs`), события различаются полем `event`. Полный разбор почему так — [PROJECT_BRIEF.md раздел 14.8](../docs/PROJECT_BRIEF.md#148-аналитика).

**Что отправляется:**

| Событие | Когда | Поле `article` |
|---|---|---|
| `просмотр_таплинк` | при загрузке страницы | `Таплинк` |
| `клик_таплинк` | при клике по любой карточке с `data-track` (без `data-event`) | значение `data-track` (slug) |
| значение `data-event` | при клике по карточке с `data-event` (перебивает дефолт) | значение `data-track` (slug) |

**Slug'и кнопок (`data-track="..."` в `index.html`):**

| Slug | Что | Где в HTML | Кастомное событие (`data-event`) |
|---|---|---|---|
| `tochka-razvorota` | Главный баннер — интенсив «Точка Разворота» | `class="banner"` | `клик_точка_разворота` |
| `igra` | Карточка «Игра, в которую играют люди» | `class="card"` | — |
| `lyubov` | Карточка «Любовь и зависимость» | `class="card"` | — |
| `mama` | Карточка «Я и мама» | `class="card"` | — |
| `professiya` | Курс «Профессия психолог» | `class="course"` | — |
| `5-klyuchey` | Курс «5 ключей к себе» | `class="course"` | — |
| `tg` | Telegram-канал | `class="channel"` | — |
| `yt` | YouTube | `class="channel"` | — |

Табы (`Тесты`/`Курсы`/`Каналы`) **не трекаются** — это UI-навигация, аналитической ценности нет.

**Где смотреть данные:** Google Sheets (тот же документ, что для статей) → лист **«Данные»**. Каждое событие = отдельная строка с временем. В листе **«Статистика»** побочно появляются 9 строк-счётчиков (`Таплинк` + 8 slug'ов) рядом со статьями. В **«Конверсия»** / **«Рейтинг»** не попадает (whitelist в Apps Script).

**Реализация:** одиночный inline-блок в `<script>` в конце `index.html`, перед IIFE для табов. Использует `fetch(URL, {keepalive: true, mode: "no-cors"})` (запрос переживает navigation) с Image-fallback. Делегированный listener на `document` в capture-фазе ловит клики до навигации.

**Как добавить новый трекаемый элемент:**
1. Повесить атрибут `data-track="<slug>"` на любой кликабельный `<a>` или родителя `<a>`.
2. Slug — короткое латинское имя в snake-case или kebab-case. Никакого кода трогать не надо — listener делегированный.
3. Если у кнопки должно быть собственное имя события в листе «Данные» (не дефолтное `клик_таплинк`) — добавить ещё `data-event="<имя_события>"`. Это нужно, например, чтобы клик по интенсиву летел как `клик_точка_разворота` отдельной строкой, а не сливался с остальными карточками.
4. После деплоя проверить в листе «Данные» что событие появилось.

**Что отслеживать НЕ надо** (по договорённости):
- Клики по табам.
- Mouse/scroll/keyboard-события — для тепловых карт и сценариев лучше Я.Метрика (см. backlog).

**При смене эндпоинта Apps Script:** обновлять константу `ENDPOINT` в `<script>` блоке `taplink/index.html` дополнительно к `shared/analytics.js` и `articles/anketa-th.html`.

## Что редактировать

### Тексты

Все тексты — прямо в `index.html`, между тегами. Семантические якоря для поиска:

| Что менять | Где искать в `index.html` |
|---|---|
| Имя в шапке | `<h1 class="header__name">` |
| 4 строки регалий | блок `<div class="header__lines">` (внутри 4× `<div class="header__line">`) |
| Названия табов | `<button class="tab" id="tab-...">` |
| Текст главного баннера | `<p class="banner__title">` (только описание, заголовок-дубль убран — он на JPEG-обложке) |
| Карточки тестов: заголовок | `<h3 class="card__title">` (внутри `class="card" data-track="igra/lyubov/mama"`) |
| Карточки тестов: мета-строка | `<div class="card__meta">` (под заголовком, разделение пунктов через ` · `) |

| Цены курсов | `class="course__price-old"` (старая) и соседний `<span>` (новая) |
| Подписи каналов | `class="channel__name"` (имя) / `class="channel__sub"` (описание под именем) |
| Превью-описание (Telegram/OG) | `<meta name="description">` и `<meta property="og:description">` в `<head>` (одинаковые) |

### Ссылки

Все href уже подставлены, см. [`LINKS.md`](./LINKS.md). **UTM-разметку не трогать** — на курсах `?utm_source=social_taplink` стоит исторически, на бот-ссылках `?start=...` — это идентификаторы воронок Telegram.

### Картинки

Если меняем — класть в `images/` с тем же именем. **Перед коммитом — пережать**, иначе перфоманс просядет:

```bash
# resize до 2× от показа, JPEG quality 82
sips -Z 960 -s formatOptions 82 taplink/images/banner-tochka-razvorota.jpg --out taplink/images/banner-tochka-razvorota.jpg

# конвертация PNG → JPG с ресайзом (если исходник в PNG)
sips -s format jpeg -Z 960 -s formatOptions 82 source.png --out taplink/images/<имя>.jpg

# или для аватара/каналов (квадраты)
sips -Z 192 -s formatOptions 80 taplink/images/avatar-ksenia.jpg --out taplink/images/avatar-ksenia.jpg
```

Целевые размеры (ширина в px после ресайза):

| Слот | Целевая ширина | Показ | Quality |
|---|---|---|---|
| Аватар | 192 (квадрат) | 96×96 | 80 |
| Главный баннер | 960 | full-width 480 | 82 |
| Карточки тестов (igra/lyubov/mama) | 320 | 112×84 | 82 |
| Курсы (professiya/5-klyuchey) | 960 | full-width 480 | 82 |
| Каналы (tg/yt) | 128 (квадрат) | 56×56 | 82 |

Aspect-ratio главного баннера задаётся CSS-правилом `.banner__cover { aspect-ratio: 3/2; }` (с 22.05.2026 — было `16/10` под исходную обложку Весны 960×600). Если меняете картинку — либо подберите 3:2, либо обновите CSS под новые пропорции.

После замены — обновить `width`/`height` в `<img>` если меняется aspect ratio.
